## 1. Rule changes (the frozen-core delta — one line per F-id)

- **F2e / F72** — publisher distinctness is computed over **collapsed** publisher identities, not raw
  netlocs. The set at `gate.py:35` (`{publisher_key(e) for e in f.evidence}`) becomes
  `{collapsed_publisher(e) for e in f.evidence}`, where `collapsed_publisher` maps (a) any netloc in
  `registry/syndicators.json` to its **originating publisher**, and (b) any evidence whose body is a
  near-duplicate of another citation's body (L1 infra) to a **single** shared identity. One wire
  story on 3 syndicator domains → **1** distinct publisher; `minDistinctPublishers: 3` therefore can
  no longer be satisfied by syndication. Error text keeps the v1.3 shape: `(K distinct publishers < N)`.
- **F71** — an **anchor-FORCED** rating move (the Part 7 bias guardrail makes the prior rating
  illegal, so `gate.py` bounds it) is **exempt from the evidence-sufficiency gate**: it is
  code-computed measured evidence, not a judgment re-rate. The exempted rating is **stamped
  `anchor-bounded on thin evidence`** (surfaced in the trust footer). A genuine qualitative re-rate on
  the same thin evidence is **still blocked** (regression guard). The whole-run `--no-sufficiency`
  flag is **removed from live paths** (its only sanctioned use — the anchor-forced deadlock — is now
  handled in code) or reduced to **per-dimension + required reason** (see §7 Q3).

## 2. Biggest behavioral delta — "the silent hole F72 closes"

Unlike v1.3 (which *loosened* F2e with a corroboration exception), v1.4's F72 half **tightens** the
same gate against a silent bypass. The current only defense against cross-domain wire syndication is
an **un-gated extraction-prompt sentence** ("distinct outlets, not syndication of one story") — the
exact "nothing un-gated reaches a number" anti-pattern the doctrine forbids. v1.4 moves that defense
into deterministic code (registry + near-dup). This is the mirror image of F71: F71 is *honest* thin
evidence failing loudly; F72 is *disguised* thin evidence (1 story, 3 netlocs) that today passes
**unremarked** — no bypass record, nothing for review to catch. Closing it also gives F69's open
handoff note a home: the gatherer records the **originating publisher** on the blob metadata (D3),
not free text in `content`.

## 3. schemaVersion decision (explicit, per the playbook)

**`schemaVersion` stays `1.2` — no schema field is added or changed.** (D3.)
- F72's originating-publisher datum lives in **gather-blob metadata** (the gatherer-contract surface),
  not on the `Finding` model.
- F71's `anchor-bounded on thin evidence` stamp rides an **existing** free-text surface (the F3
  `dimensionStatus` note / confidence-cap note that already carries strings like
  `"secondary-only evidence"`), **not** a new field — see §7 Q2 to confirm the exact carrier.

## 4. Deliberately unchanged (what a reader might assume moved but didn't)

- **Scoring math** — DMI/SMI/anchors/divergence are byte-identical. F71/F72 are gate/corroboration
  semantics; **no scoring indicator, weight, or side-rule changes.** Therefore `fixtures/golden/`
  (`anchors/findings/ratings/scorecard/status.json`) is **regenerated NOT AT ALL** — there is no math
  delta to re-pin. (Contrast v1.2, which regenerated goldens with a hand computation.)
- **The F63 corroboration exception itself** (all-secondary can be `high` at ≥N publishers) — v1.4
  changes only *how publishers are counted*, not the exception's existence.
- **`minDistinctPublishers: 3`** — the threshold is unchanged; only the denominator (distinct-count)
  becomes syndication-resistant.
- **The Finding schema, judge aggregation, pipeline, JsonStore** — byte-identical to v1.3.
- **F3 dimension-level confidence-cap badge** — unchanged (same carve-out v1.3 named).

## 5. Shadow-run (gate-behavior, NO store writes) — mandatory

Because F72 changes a computed gate result, quantify the blast radius the way v1.2's honesty section
did — but over **gate decisions**, not scoring:

- `scripts/shadow_run_v14.py` (no store writes): for every stored finding in
  `store/chips.merchant-gpu/*.json`, compute the F2e distinct-publisher count **old** (raw netloc)
  vs **new** (collapsed), and the F71 precedence outcome **old** (deadlock/bypass) vs **new**
  (exemption path), and emit a **flip table** — every finding whose corroboration verdict changes,
  attributed to the syndicator-registry rule or the near-dup rule.
- **Honesty section in the migration doc:** state plainly whether any *currently-stored*
  high-confidence finding, thesis reversal, or wiki promotion **relied on** the syndication hole
  (i.e., would flip from pass→fail under v1.4). Per the campaign's Phase-3 note, this is a *proven
  structural risk not yet a witnessed live failure* — the shadow-run confirms which it is **today**.

## 6. Replay decision — recommend NONE (follow v1.3), retro-note instead

- **No replay / no re-gating of stored scorecards.** Following the v1.3 precedent (a gate-rule change
  with no scoring-math delta has nothing to replay) and charter Part 33 (originals immutable), the
  stored series stays **byte-unchanged**. We do **not** re-run history through the new gate — that
  would rewrite append-only records.
- **If §5's shadow-run shows a stored scorecard relied on the hole:** disclose it via a **retro
  trust-footer note** on the next render (the same surface F75 uses for the 2026-07-v3 sufficiency
  bypass), **not** by mutating the store. The render is the correct surface for hindsight; the
  scorecard is not. → §7 Q4 asks the user to confirm this over a replay.

## 7. Open decisions requiring the user's sign-off (frozen-core — never AFK)

1. **`sufficiency.py` frozen-core status.** It is **not** on `docs/roadmap.md`'s formal Standing
   Constraints (frozen-core) list, yet F71's fix touches gate semantics as directly as `gate.py`.
   **Decide:** (i) add `sufficiency.py` to the formal frozen-core list via this migration, or (ii)
   treat it as frozen-core-*adjacent* for v1.4 without amending the list. Recommendation: **(i)** —
   the exemption logic is gate semantics; make the list honest.
2. **F71 stamp carrier.** Confirm `anchor-bounded on thin evidence` rides the existing
   `dimensionStatus` note / confidence-cap surface (no schema bump, honoring D3) rather than a new
   field. Recommendation: **existing note surface.**
3. **`--no-sufficiency` — remove vs per-dimension.** Once the exemption path exists, the flag's only
   sanctioned use is gone. **Decide:** remove it entirely, or keep it **per-dimension + required
   reason + logged** (F75's general waiver shape) for non-anchor deadlocks. Recommendation: **remove
   the whole-run flag; let F75's per-item waiver cover the residual**, so no whole-run bypass survives
   before the unattended pilot.
4. **Retro trust-footer note vs replay (from §6).** Confirm: disclose any historical reliance on the
   F72 hole via a render-time retro note, **not** by appending re-gated scorecard versions.
   Recommendation: **retro note.**
5. **Prompt-change / eval coupling.** F72 moves the syndication defense into code; the extraction
   prompt sentence can then **stay as-is** (now enforced by the gate) → **no hash-pin trip, no
   run-eval needed**. **Decide:** leave the prompt unchanged (recommended — keeps v1.4 a pure
   gate+data migration), or *also* tighten the prompt wording (which trips the pin and pulls in the
   full `run-eval` governance: Opus brains + Opus graders, replicate baseline). Recommendation:
   **leave the prompt unchanged.**
6. **F72(b) near-dup mechanism depth.** `dedup.py:53 content_hash` is an **exact** hash; wire reprints
   are often near-identical (different site wrappers), not byte-identical. **Decide:** ship v1.4 with
   (i) exact-hash collapse + the syndicator registry (registry catches the wrapper-different reprints
   by netloc), deferring shingle/normalized similarity to a follow-up; or (ii) build normalized/
   shingle near-dup now. Recommendation: **(i)** — the registry (D5a) covers the common
   wrapper-different case deterministically; a normalized-similarity pass is a bounded follow-up if
   the shadow-run shows unknown-syndicator reprints slipping through.
